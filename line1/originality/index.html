Computable differentiation of the HITC FHIR methodology from established FHIR base capabilities

Mechanism Comparison

The table below maps each established FHIR mechanism to its usage within this project and documents the specific contribution or extension that distinguishes this work from standard FHIR base capabilities.

Established FHIR Mechanism Usage in This Project Contribution
StructureDefinition Constrains FHIR resources for clinical data exchange Forms cross-format canonical model set (35 profiles)
StructureMap Instance-level transformation between source and target Composes into a documented two-phase mapping pipeline with ConceptMap-driven semantic reconciliation
ConceptMap Declares terminology relationships Bridges structural mapping and semantic reconciliation with confidence/reviewer metadata
Observation Represents clinical measurements Repurposed to express computable exchange-quality metrics (6 dimensions)
Provenance Records resource origin and transformation history Extends to record resource-level transformation lineage with Merkle-root tamper evidence
Merkle Tree Integrity verification of resource bundles Linked to Provenance transformation records for auditable data quality assertions
OperationDefinition Declares custom FHIR operations Establishes bundle validation and federated summary contracts for exchange governance

Conclusion

"The contribution lies in the particular computable combination, constraint architecture, workflow, and application of established FHIR mechanisms—not in claiming invention of the underlying standards."
